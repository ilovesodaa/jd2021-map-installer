# Original JDU Data Usage vs Extracted Map Builder Report

Based on the analysis of the `map_builder.py` / `map_installer.py` scripts and cross-referencing against a "complete" standard custom map like `GetGetDown`, here is a breakdown of what original extracted data your scripts actually process, versus what is completely disregarded.

## ✅ Information Used (Extracted & Processed)

The scripts are highly targeted and extract only the bare minimum required to make a map playable with a video background. 

1. **Audio/Music Track Metadata**
   - **Source:** `*musictrack.tpl.ckd` (Extracted JSON)
   - **Data Preserved:** Beat grid/markers, signatures, song sections, `startBeat`, `endBeat`, `videoStartTime`, `previewEntry`, and fade parameters.
   - **Result:** Rebuilt into a functional `<map_name>.trk` file used by the engine for timing gameplay.
2. **Song Description Metadata**
   - **Source:** `*songdesc.tpl.ckd` (Extracted JSON)
   - **Data Preserved:** Artist, Title, Credits, Difficulty, SweatDifficulty.
   - **Result:** Used to generate the `SongDesc.tpl` file, which is responsible for the song's appearance in the main menu and coverflow.
3. **Menu Art & Textures**
   - **Source:** `*.png`, `*.jpg`, and decoded `*.tga.ckd` files from the downloaded archives.
   - **Data Preserved:** Coach phone images (`_coach_*_phone`), Album Cover elements (`cover_generic`, `cover_online`, `banner_bkg`, `cover_albumbkg`), and Map Background (`map_bkg`).
   - **Result:** Placed directly into `MenuArt/textures/` and linked via dummy Actor Templates generated by the script.
4. **Choreography Timelines & Pictograms**
   - **Source:** `*_tml_dance.?tape.ckd`, `*_tml_karaoke.?tape.ckd`, and `pictos/*.png.ckd`
   - **Data Preserved:** The JSON movement/lyric data and pictogram textures.
   - **Result:** Converted perfectly using `json_to_lua.py` into usable `.dtape`/`.ktape` formats, mapping the coach moves and UI lyrics.
5. **Video & Main Audio Media**
   - **Source:** The manually downloaded `.webm` (Video) and `.ogg` (Audio).
   - **Data Preserved:** The core video/audio. The scripts handle FFmpeg offset trimming for the gameplay `.wav`.

## ❌ Information Disregarded or Missing Entirely

Your map builder discards almost all "engine-specific" logic elements from the original server files. A huge part of this is because **the script ONLY downloads the `MAIN_SCENE_*.zip` package** and ignores the platform-specific scene packages where crucial gameplay mechanics are stored.

1. **Gestures & Scoring Data (`Timeline/Moves/` folder) 🔴 (Critical Missing Data)**
   - **Missing Files:** The `.msm` (gesture mappings), `.msq` (scoring sequence logic), and gesture `.msh` models.
   - **The Issue:** The `map_builder.py` script has no logic to handle the `Moves/` folder. More importantly, real JDU scoring data for specific controllers isn't even in the `MAIN_SCENE` package you are downloading. Controller/Joycon moves are traditionally stored in the `WII_SCENE` or `ORBIS_SCENE` download packages, while camera tracking moves are in the `KINECT_SCENE` or `DURANGO_SCENE`.
   - **Consequence:** Because you aren't downloading the controller-specific logic packages or injecting the timeline tracking data for them, the game engine defaults to generic placeholder movements (which means the player cannot score properly on specific moves—it just gives out "OK" or "Good" randomly based on standard controller motion).

2. **Autodance Data (`Autodance/` folder) 🔴 (The critical missing visual piece)**
   - **Ignored Files:** `.adrecording`, `.adtape`, `.advideo`, `_AUTODANCE.isc`, `_Autodance.act`, and `_Autodance.tpl`.
   - **The Issue:** The script never loops over or generates Autodance config files. More importantly, when it generates the overarching `_MAIN_SCENE.isc`, it explicitly writes dummy entries for `AUDIO`, `CINE`, `TML`, and `VIDEO`, but **completely excludes the `Autodance` SubSceneActor.**
   - **Consequence:** The game doesn't know *when* or *how* to capture camera clips for the map, so the feature doesn't work.

3. **Cinematics Data (`Cinematics/` folder)**
   - **Ignored Files:** The true `_MainSequence.tape`.
   - **What happens instead:** The script hardcodes a completely blank dummy tape (`TapeBarCount = 1`, `Clips = {}`). 
   - **Consequence:** Any custom camera movements, advanced lighting effects, post-processing filters, or Soundwich UI events present in the original JDU map are ignored.

4. **Background Sequences & Shaders (`Background/` folder)**
   - **Ignored Files:** Original `.btape`, `.bg.act`, shaders, background models.
   - **What happens instead:** Since the map relies on a flat `.webm` video background playing behind the UI, these assets aren't necessary for the visual output. However, if the JDU map had 3D elements dynamically reacting to the player, they are lost.

5. **Graph State Machine Logic (`Graph/` folder)**
   - **Ignored Files:** Custom `.isc` state graphs. 
   - **Consequence:** Some maps use graph logic for unique intro/outro transitions. The script replaces this with the generic video player logic.

6. **Timeline - Specialized Tapes**
   - **Ignored Files:** `Stereo.stape` / Audio Sequence tapes.
   - **What happens instead:** The script generates a basic `_sequence.tpl` with a generic `.stape` implementation. Ambience/crowd `.atape` files present in older maps are completely undocumented by the tool.

### 💡 Next Steps to Fix Gestures and Autodance
If you want to fully flesh out this tool:
1. **For Gestures/Scoring:** You need to update `asset_mapping.html` / `map_downloader.py` to also locate and download the `[MAPNAME]_WII_SCENE` or `[MAPNAME]_NX_SCENE` zip files from the JDU server. Once downloaded, `map_builder.py` needs to extract the `.msm` and `.msq` files into `Timeline/Moves/` and generate the appropriate Timeline `.isc` bindings.
2. **For Autodance:** You need to parse `.adtape`, `.adrecording`, and `.advideo` from the `MAIN_SCENE` `.ckd` files. You must also inject an `<ACTORS NAME="SubSceneActor">` for `Autodance` inside the generated `_MAIN_SCENE.isc` linked to an `_AUTODANCE.isc`.
