# JDNext Map Source -> JD2021 Output Mapping

Last updated: 2026-04-06

This document maps JDNext mapPackage source data (mainly MonoBehaviour JSON + extracted assets)
into the JD2021 files the game expects under World/MAPS/<codename>/.

## Scope and evidence

Primary observed source sample:
- temp/_extraction/SweetButPsycho/monobehaviour/map.json
- temp/_extraction/SweetButPsycho/monobehaviour/musictrack.json
- temp/_extraction/SweetButPsycho/assets.html

Relevant pipeline code paths:
- jd2021_installer/extractors/jdnext_bundle_strategy.py
- jd2021_installer/parsers/normalizer.py
- jd2021_installer/installers/game_writer.py

## High-level source payloads

JDNext mapPackage extraction currently yields these useful payloads:
- monobehaviour/map.json
- monobehaviour/musictrack.json
- textasset/*.gesture
- textasset/*.msm
- texture/sprite PNG assets (pictos + menu art)
- downloaded media links and metadata from assets.html (title/artist and URLs)

## Required JD2021 outputs and source mapping

### 1) Song metadata

Target files:
- SongDesc.tpl
- SongDesc.act

Target fields in SongDesc.tpl are written from SongDescription dataclass
(game_writer._write_songdesc).

Source mapping status:

| SongDesc.tpl field | Preferred JDNext source | Current status |
|---|---|---|
| MapName | map.json -> SongDesc.MapName or map.json -> MapName | Partially mapped (today uses codename/songdesc CKD path logic) |
| JDVersion | map.json -> SongDesc.JDVersion | Not consumed from map.json today |
| OriginalJDVersion | map.json -> SongDesc.OriginalJDVersion | Not consumed from map.json today |
| Artist | map.json -> SongDesc.Artist | Fallback from assets.html when songdesc CKD missing |
| DancerName | map.json -> SongDesc.DancerName | Not consumed from map.json today |
| Title | map.json -> SongDesc.Title | Fallback from assets.html when songdesc CKD missing |
| Credits | map.json -> SongDesc.Credits | Not consumed from map.json today |
| NumCoach | map.json -> SongDesc.NumCoach; fallback from discovered coach media count | Media fallback exists; map.json SongDesc not used |
| MainCoach | map.json -> SongDesc.MainCoach | Not consumed from map.json today |
| Difficulty | map.json -> SongDesc.Difficulty | Not consumed from map.json today |
| SweatDifficulty | map.json -> SongDesc.SweatDifficulty | Not consumed from map.json today |
| LyricsType | no clear field in observed map.json sample | Defaults/fallbacks only |
| backgroundType | no clear field in observed map.json sample | Defaults/fallbacks only |
| Tags | no clear field in observed map.json sample | Defaults/fallbacks only |
| Energy | no clear field in observed map.json sample | Defaults/fallbacks only |
| LocaleID / VersionLocId / MojoValue | no clear field in observed map.json sample | Defaults/fallbacks only |
| DefaultColors | no clear field in observed map.json sample | Defaults/fallbacks only |
| PhoneImages | inferred by convention from codename | Generated by fallback pattern |

Current behavior summary:
- If no *songdesc*.tpl.ckd exists, normalizer._extract_song_desc falls back to assets.html only
  for Title + Artist and leaves the rest as defaults.
- map.json SongDesc block is not currently consumed by normalizer._extract_song_desc.

Recommendation:
- Add map.json SongDesc as fallback tier between songdesc CKD and assets.html.

Proposed priority for SongDesc data:
1. songdesc.tpl.ckd (authoritative when present)
2. monobehaviour/map.json -> SongDesc
3. assets.html (title/artist only)
4. hardcoded defaults

### 2) Music timing and preview

Target files:
- Audio/<codename>.trk
- Audio/<codename>_musictrack.tpl

Source mapping:

| Target field | JDNext source |
|---|---|
| markers | musictrack.json -> m_structure.MusicTrackStructure.markers[].VAL |
| signatures | musictrack.json -> ...signatures[].MusicSignature |
| sections | musictrack.json -> ...sections[].MusicSection |
| startBeat/endBeat | musictrack.json -> ...startBeat/endBeat |
| videoStartTime | musictrack.json -> ...videoStartTime |
| previewEntry/previewLoopStart/previewLoopEnd | musictrack.json -> corresponding fields |
| volume/fadeIn/fadeOut | musictrack.json -> corresponding fields |

Current behavior summary:
- jdnext_bundle_strategy synthesizes <codename>_musictrack.tpl.ckd JSON from musictrack.json.
- normalizer parses synthesized CKD -> MusicTrackStructure.
- game_writer writes .trk from normalized MusicTrackStructure.

This path is already good and complete for the observed JDNext sample.

### 3) Dance choreography timeline

Target files:
- Timeline/<codename>_TML_Dance.dtape
- Timeline/<codename>_TML_Dance.tpl
- Timeline/<codename>_TML_Dance.act

Source mapping:

| Target clip class | JDNext source |
|---|---|
| MotionClip | map.json -> DanceData.MotionClips[] |
| PictogramClip | map.json -> DanceData.PictoClips[] |
| GoldEffectClip | map.json -> DanceData.GoldEffectClips[] |

Field mapping already implemented in jdnext_bundle_strategy._synthesize_tapes_from_map_json:
- StartTime, Duration, Id, TrackId, IsActive
- MotionClip: MoveName, GoldMove, CoachId, MoveType, Color
- PictogramClip: PictoPath, CoachCount
- GoldEffectClip: EffectType

Source mapping:

### 8) Media streams

Target files:

Source mapping:
- assets.html provides downloadable URL variants for video tiers and audio.
- extractor picks quality variant and writes media into extraction directory.

## Known gaps specific to JDNext SongDesc parity

The observed JDNext map source contains a SongDesc object in map.json, but current
normalization does not consume it as a fallback. This causes these avoidable losses:
- Difficulty and SweatDifficulty can fall back to defaults even when source has values.
- DancerName/Credits/MainCoach/JDVersion may be lost.
- NumCoach may rely on media inference instead of explicit source metadata.

## Proposed implementation checklist

1. Add helper in normalizer to discover monobehaviour/map.json for codename.
2. Parse map.json SongDesc block into SongDescription when songdesc CKD is absent.
3. Keep existing assets.html fallback for blank title/artist fields.
4. Continue media-based coach count correction as final post-step.
5. Add tests for:
   - map.json SongDesc fallback used when songdesc CKD missing
   - empty Artist/Title in map.json falls back to assets.html
   - non-empty Difficulty/NumCoach/MainCoach survive to generated SongDesc.tpl
